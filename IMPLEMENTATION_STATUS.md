# 実装状況管理

最終更新: 2026-04-29

## 実装済み
- 共通レイアウト（左メニュー開閉、Fキー2回トグル、一覧の詳細ボタンで詳細画面表示；**トップバーにログイン中表示とログイン切替**）。**メイン領域のリスト/詳細は白い角丸台座なし**（ページ背景に馴染むフラットレイアウト）
- **ログイン**（`/login`、`POST /api/auth/login` で ID・パスワード検証、`sha256:` 照合、シード直後のプレースホルダハッシュ救済あり、`tei_user_name` 保持、未ログインは `/login` へ；**画面に初期アカウントの説明は表示しない**）。**自分の氏名・ログインID・パスワード**はトップバー「**アカウント設定**」（`/account`）や **`GET/PATCH /api/auth/me`**（変更時は現在のパスワード必須、**社員管理権限不要**；`PATCH` の本文 `userName` で氏名）。**初期管理者のシード表示名は Admin**（003・**009**）
- **一覧の簡易検索**（表示データに対する部分一致フィルタ：案件・顧客・納品先・見積・受注・小口受注・型工務・請求・税率・社員・抜き型/LC。検索・新規作成は**白いリスト枠の外**の見出し行に配置）
- 案件管理（一覧、`/cases/new` で新規作成、`/cases/[caseId]` で別URLの詳細＋**下線タブ**の4タブ（見積・小口・受注・抜き型/LC）；**営業担当（ユーザ）・案件種別のプルダウン**、顧客拠点/顧客担当者は **UUID手入力** で紐付け）
- 案件参照API：`GET /api/users`（有効ユーザ一覧・案件メニュー閲覧権限）、`GET /api/case-types`（案件種別）
- 見積・受注・小口受注・抜き型/LC：URLクエリ `caseId` で案件プリ選択（案件詳細タブからの遷移に対応）
- **新規レコード作成の導線**：主要メニューすべて一覧の「新規作成」から **`/new` 専用画面**へ遷移（案件 `/cases/new`、顧客 `/customers/new`、納品先 `/delivery-destinations/new`、見積 `/estimates/new`、受注 `/orders/new`、小口 `/small-orders/new`、請求 `/invoices/new`、型工務 `/manufacturing/new`、税率 `/masters/new`、抜き型/LC `/specs/new`）。`caseId` / `orderId` はクエリで引き継ぎ可能
- 請求・型工務一覧：受注切替時に URL `orderId` を同期
- 顧客管理（団体・部署・有効フラグの更新；**一覧の新規は `/customers/new`**。詳細は**下線タブ**（基本・拠点・担当者・請求・納品先）。詳細内の拠点・担当者・請求・納品先の追加・編集は各タブのまま）
- 納品先管理（一覧/詳細、**新規は `/delivery-destinations/new`**）
- 見積（一覧/詳細、**新規は `/estimates/new`**、明細追加、合計表示）
- 受注（一覧/詳細、**新規は `/orders/new`**、明細追加、合計表示）
- 小口受注（一覧/詳細、**新規は `/small-orders/new`**、ヘッダのDB全項目、明細の追加・行保存）
- 型工務（一覧/詳細、**新規は `/manufacturing/new`**、進捗フラグ更新）
- 抜き型/LC仕様（一覧・受注選択、**新規は `/specs/new`**；既存行の全項目編集・保存、JSONはテキストエリア）
- 請求（一覧/詳細、**新規は `/invoices/new`**）
- 税率マスタ（一覧/詳細、**新規は `/masters/new`**）
- **社員管理**（左メニュー `/staff`、一覧→詳細編集、**新規は `/staff/new`（初期パスワード必須）**、詳細で**新パスワード**変更可）。API：`GET/POST /api/staff`、`PATCH /api/staff/[id]`（任意 `newPassword`）、`GET /api/user-groups`。管理者グループの **`staff` メニュー権限**は 003・006・**008** で投入
- RBAC（`x-login-id` + `roles`判定）
- 監査ログ（トリガー + 2週間保持関数）
- 楽観的ロック（主要更新系APIで`version`判定）
